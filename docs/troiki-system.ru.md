# Система Тройки

Тройки — стратегическая система управления задачами, основанная на ограничении числом 3.

## Структура

Единица работы — **проект**. Каждый проект может быть назначен в одну из трёх категорий, в каждой не более 3 проектов (все задачи назначенного проекта, включая подзадачи, отображаются вместе):

| Категория  | Описание                                                   |
|------------|------------------------------------------------------------|
| Важное     | Самые приоритетные проекты. Всегда доступна.               |
| Среднее    | Разблокируется выполнением задач в проектах из Важного.    |
| Остальное  | Разблокируется выполнением задач в проектах из Среднего.   |

При назначении проекта в категорию приоритет всех его открытых задач (root + подзадачи) форсится в derived priority категории. Ручное редактирование приоритета отклоняется, пока у проекта есть категория.

## Правила

1. **Важное** — проекты можно назначать в любой момент, пока их менее 3.
2. **Среднее** — каждое завершение задачи в проекте из Важного открывает +1 слот в Среднем.
3. **Остальное** — каждое завершение задачи в проекте из Среднего открывает +1 слот в Остальном.
4. Важное **не влияет** на Остальное напрямую. Цепочка: Важное → Среднее → Остальное.
5. Снятие проекта с категории освобождает слот, но **не открывает** слот в следующей категории — только завершение.
6. Открытые слоты (capacity) накапливаются и не сгорают.
7. Гранты capacity идемпотентны на уровне задачи (`tasks.troiki_capacity_granted`) — повторное выполнение задачи не даёт дополнительного слота.

## Начальное состояние (initial fill)

До нажатия «Запустить систему» все три категории доступны для предварительного заполнения, при этом универсальный лимит ≤ 3 проектов на категорию соблюдается.

- Важное: capacity = 3.
- Среднее и Остальное: capacity = 3 (initial-fill).

После старта Среднее/Остальное переходят в режим накопления: capacity снимается со снапшота текущего числа проектов в категории и в дальнейшем растёт только за счёт завершения задач в проектах из вышестоящей категории.

## Рекомендации

- Формулируйте задачи как конкретные действия (actionable tasks) по GTD.
- После выполнения задачи ставьте следующую, чтобы проект продолжал двигаться.

## Сброс

Кнопка **Сбросить систему** (показывается на месте «Запустить», когда цикл уже идёт) возвращает Тройку в исходное состояние. После подтверждения она:

- Снимает категорию со всех проектов (`projects.troiki_category = NULL`).
- Сбрасывает флаг выдачи слотов на задачах (`tasks.troiki_capacity_granted = 0`), чтобы те же задачи могли участвовать в новом цикле.
- Обнуляет накопленные слоты Medium и Rest (`users.troiki_medium_capacity = 0`, `troiki_rest_capacity = 0`).
- Возвращает флаг цикла в исходное (`users.troiki_started = 0`).

Приоритеты задач **не трогаются** — сброс очищает *цикл*, а не backlog пользователя. Операция идемпотентна: повторный Reset на уже сброшенной системе ничего не меняет.
