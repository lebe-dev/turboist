# Система Тройки

Тройки — стратегическая система управления задачами, основанная на ограничении числом 3.

## Структура

Три категории, каждая вмещает максимум 3 корневых задачи (количество подзадач не ограничено):

| Категория  | Описание                                    |
|------------|---------------------------------------------|
| Важное     | Самые приоритетные задачи. Всегда доступна.  |
| Среднее    | Разблокируется выполнением задач из Важного. |
| Остальное  | Разблокируется выполнением задач из Среднего.|

## Правила

1. **Важное** — можно добавлять задачи в любой момент, пока их менее 3.
2. **Среднее** — каждое завершение задачи в Важном открывает +1 слот в Среднем.
3. **Остальное** — каждое завершение задачи в Среднем открывает +1 слот в Остальном.
4. Важное **не влияет** на Остальное напрямую. Цепочка: Важное → Среднее → Остальное.
5. Удаление задачи освобождает слот, но **не открывает** слот в следующей категории — только завершение.
6. Открытые слоты (capacity) накапливаются и не сгорают.

## Начальное состояние

- Важное: доступно сразу (capacity = 3).
- Среднее и Остальное: заблокированы (capacity = 0). Для разблокировки нужно завершать задачи.
