# Система Тройки

Тройки — стратегическая система управления задачами, основанная на ограничении числом 3.

## Структура

Единица работы — **проект**. Каждый проект может быть назначен в одну из трёх категорий, в каждой не более 3 проектов (все задачи назначенного проекта, включая подзадачи, отображаются вместе):

| Категория  | Описание                                                   |
|------------|------------------------------------------------------------|
| Важное     | Самые приоритетные проекты. Всегда доступна.               |
| Среднее    | Разблокируется выполнением задач в проектах из Важного.    |
| Остальное  | Разблокируется выполнением задач в проектах из Среднего.   |

При назначении проекта в категорию приоритет всех его открытых задач (root + подзадачи) форсится в derived priority категории. Ручное редактирование приоритета отклоняется, пока у проекта есть категория.

## Правила

1. **Важное** — проекты можно назначать в любой момент, пока их менее 3.
2. **Среднее** — каждое завершение задачи в проекте из Важного открывает +1 слот в Среднем.
3. **Остальное** — каждое завершение задачи в проекте из Среднего открывает +1 слот в Остальном.
4. Важное **не влияет** на Остальное напрямую. Цепочка: Важное → Среднее → Остальное.
5. Снятие проекта с категории освобождает слот, но **не открывает** слот в следующей категории — только завершение.
6. Открытые слоты (capacity) накапливаются и не сгорают.
7. Гранты capacity идемпотентны на уровне задачи (`tasks.troiki_capacity_granted`) — повторное выполнение задачи не даёт дополнительного слота.

## Начальное состояние

- Важное: доступно сразу (capacity = 3).
- Среднее и Остальное: заблокированы (capacity = 0). Для разблокировки завершайте задачи в проектах из вышестоящих категорий.
